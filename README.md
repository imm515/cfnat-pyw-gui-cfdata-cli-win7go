# cfnat 本地订阅生成器 GUI

一个面向 Windows 的 `tkinter` 图形界面工具，用来接管 `cfnat` 的扫描输出，自动挑出可用 IP，生成本地订阅，并提供可直接给客户端使用的 HTTP 订阅地址。

适合这类场景：

- 你已经在用 `cfnat`，但不想一直盯命令行
- 你希望把优选结果自动写进节点模板
- 你希望本地始终有一个固定订阅地址可供客户端读取
- 你希望在 GUI 里看到扫描进度、IP 切换和订阅状态

## 这工具的特点

- 实时监控 `cfnat` 输出并提取有效 IP
- 自动基于模板节点生成 `subscription.txt`
- 提供本地 HTTP 订阅地址，默认 `http://127.0.0.1:8888/sub`
- 支持历史订阅恢复，重启 GUI 后仍可继续提供订阅
- 提供“测速当前订阅”按钮，单独验证当前订阅 IP
- 支持手动指定 IP 启动
- 自动记录关键日志，方便排查切换和稳定性问题

## 适合谁用

- 想把 `cfnat` 扫描结果稳定转成订阅的人
- 想减少手工替换节点 IP 的人
- 想在 Windows 7/10 上直接双击运行 GUI 的人

## 快速开始

准备好这些文件：

- `cfnat_sub_gui.pyw`
- `cfdata-cli-win7-experimental.exe`
- `nodes.txt`

启动方式：

```text
双击 cfnat_sub_gui.pyw
```

或：

```bash
python cfnat_sub_gui.pyw
```

启动后基本流程是：

1. 准备好 `nodes.txt` 节点模板
2. 在 GUI 里设置机房、延迟阈值、并发等参数
3. 点击“启动扫描”
4. 等待生成本地订阅
5. 把 `http://127.0.0.1:8888/sub` 填进客户端

## 主要功能

### 1. 扫描结果自动转订阅

- 监听 `cfnat` 输出
- 自动提取有效 IP
- 自动替换节点模板里的目标地址
- 自动生成 `subscription.txt`

### 2. 本地订阅服务

- 默认监听 `8888` 端口
- 默认地址：`http://127.0.0.1:8888/sub`
- 有历史订阅时，重启后也可直接提供订阅

### 3. 当前订阅测速

GUI 中提供“测速当前订阅”按钮，用于对当前订阅 IP 做一次独立测速。

特点：

- 首次运行但已存在历史 `subscription.txt` 时，也可直接测速
- 测速结果保留在日志区，不会被后续 `IP刷新` 覆盖
- 固定使用 `10MB` 下载文件
- 结果显示为 `MB/s`

### 4. 稳定性控制

- 默认延迟阈值为 `300ms`
- 默认优先建议有效 IP 数量最多的 colo
- 订阅 IP 达到稳定刷新次数后再切换，减少频繁波动
- 发现异常切换或 IP 池耗尽时会给出提示

## 测速原理

测速不会走系统代理或环境变量代理。

实现方式：

- 使用 Python 标准库 `socket + ssl`
- 直接连接当前订阅对应的目标 IP 和端口
- HTTPS 场景下保留目标站点的 `Host/SNI`
- 按时间窗口统计下载速度并输出结果

这更接近“当前订阅 IP 直连测速目标时的可用吞吐”，不等同于完整代理链路带宽。

## 节点模板

`nodes.txt` 支持常见公开格式，例如：

- `vless://...`
- `vmess://...`

程序会按当前优选结果替换目标节点地址，并生成新的订阅内容。

## 运行中会产生的文件

- `subscription.txt`
- `cfnat_sub.pid`
- `logs/` 下的日志文件

## 常见问题

### 启动后没有显示订阅地址？

先检查：

- `nodes.txt` 是否存在
- 本地端口是否被占用
- `cfdata-cli-win7-experimental.exe` 是否和 GUI 在同目录

### 扫描很久没有有效 IP？

可以尝试：

- 适当提高延迟阈值
- 更换机房代码
- 降低筛选条件，先验证链路是否能跑通

### 订阅 IP 切换太频繁？

程序本身已经有稳定策略，但你也可以：

- 适当放宽延迟阈值
- 观察日志里某个 colo 是否更稳定
- 等待订阅进入稳定刷新阶段后再长期使用
