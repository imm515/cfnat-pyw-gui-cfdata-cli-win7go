# cfnat 本地订阅生成器 - GUI 版本

基于 tkinter 的图形化界面版本，实时监控 cfnat 输出，自动提取优选 IP 并生成订阅地址。

## 功能特性

### 核心功能
- ✅ 实时监控 cfnat 输出，自动提取优选 IP
- ✅ 替换节点模板中 443 端口的 IP 地址
- ✅ 生成本地订阅文件（`subscription.txt`）
- ✅ 提供 HTTP 订阅服务（默认端口 8888）
- ✅ 订阅内容自动缓存，启动即可使用

### 智能优化
- 🎯 **高峰期检测**：自动识别晚高峰时段（20:00-01:00）
- 🎯 **智能建议**：根据历史数据推荐最优机房和延迟阈值
- 🎯 **自动降级**：IP 库耗尽时自动降级重试
- 🎯 **订阅锁定**：IP 刷新 10 次后自动锁定，避免频繁切换
- 🎯 **异常检测**：IP 切换过于频繁时自动警告

### 界面功能
- 📊 实时扫描进度显示（百分比、速度、剩余时间）
- 📊 机场 IP 分布统计（≥10 个 IP 的机场）
- 📊 IP 刷新次数和延迟追踪
- 📊 订阅 IP 稳定性监控
- 📊 自动保存 cfnat 日志（默认开启，智能精简）

## 快速开始

### 前置要求

- Python 3.6+（需包含 tkinter）
- cfnat 可执行文件（`cfnat-windows7-amd64.exe`）
- 节点模板文件（`nodes.txt`）

### 启动方式

#### 方式 1：双击启动（推荐）
```
双击 cfnat_sub_gui.pyw
```

#### 方式 2：命令行启动
```bash
python cfnat_sub_gui.pyw
```

#### 方式 3：命令行模式
```bash
python cfnat_sub_gui.pyw --colo HKG --delay 200 --task 100 --num 3 --ipnum 20
```

## 界面说明

### 智能配置提示区
- **时间检测**：显示当前是否处于晚高峰时段
- **建议延迟**：根据高峰期自动建议延迟阈值（晚高峰 200ms，非高峰 200ms）
- **建议机房**：根据历史扫描数据推荐延迟最低的机房
- **上次扫描统计**：显示上次扫描中低延迟机场的分布

### 扫描配置区
| 参数 | 说明 | 默认值 | 推荐值 |
|------|------|--------|--------|
| 机房代码 | cfnat 扫描的目标机房 | HKG | 根据建议选择 |
| 延迟阈值(ms) | IP 延迟筛选阈值 | 200 | 晚高峰 200-300，非高峰 100-200 |
| 并发任务 | 同时扫描的并发数 | 100 | 50-150 |
| 负载数 | 每个 IP 的连接数 | 3 | 3-5 |
| 提取IP数 | 提取的有效IP数量 | 20 | 20-50 |
| 服务端口 | 订阅服务监听端口 | 8888 | 自定义 |

**参数详细说明**:
- **机房代码**: 指定 Cloudflare 数据中心，不同机房延迟不同
- **延迟阈值**: 只选择延迟低于此值的 IP，值越小筛选越严格
- **并发任务**: 控制扫描速度，值越大扫描越快但 CPU 占用越高
- **负载数**: 每个 IP 的并行连接数，影响负载均衡
- **提取IP数**: 从扫描结果中提取的 IP 数量，决定 IP 池大小



### 控制按钮
- **启动扫描**：启动 cfnat 扫描和订阅服务
- **重启扫描**：重启扫描（保留日志）
- **停止扫描**：停止 cfnat 进程
- **保存cfnat日志**：默认开启，智能精简记录关键信息到 `logs/` 目录

### 日志显示区
实时显示扫描进度、IP 切换、订阅更新等信息，包括：
- 扫描进度（百分比、速度、剩余时间）
- 各机场 IP 分布统计
- IP 切换历史和刷新次数
- 订阅 IP 更新状态

## 使用流程

1. **准备节点模板**
   创建 `nodes.txt` 文件，添加你的节点链接（支持 vmess、vless 格式）
   ```
   vmess://...
   vless://...
   ```

2. **启动程序**
   双击 `cfnat_sub_gui.pyw` 或命令行运行

3. **配置参数**
   - 查看智能建议，调整参数
   - 选择机房代码和延迟阈值

4. **启动扫描**
   点击"启动扫描"按钮，等待扫描完成

5. **使用订阅**
   将显示的订阅地址复制到代理客户端：
   ```
   本地访问:   http://127.0.0.1:8888/sub
   局域网访问: http://192.168.x.x:8888/sub
   ```

## 高级功能

### 订阅 IP 锁定策略
- 当某个 IP 刷新次数达到 **10 次**后，自动锁定该 IP 为订阅 IP
- 锁定后订阅内容不再切换，保持稳定性
- 减少代理客户端频繁更新节点列表

### 高峰期自动降级
- 检测到 IP 库频繁耗尽时（10 分钟内 2 次）
- 自动提高延迟阈值 +100ms（最高 300ms）
- 自动切换到延迟更低的机房
- 仅触发 1 次，避免无限降级

### IP 频繁切换警告
- 非高峰期检测到连续 2 次 IP 切换间隔 < 10 分钟
- 自动停止扫描并发出警告
- 提示检查网络或 cfnat 配置

### cfnat 日志保存
- 默认自动保存 cfnat 日志到 `logs/cfnat_YYYYMMDD_HHMMSS.log`
- 智能精简：只记录扫描进度关键节点（0%、25%、50%、75%、100%）和前3个有效IP
- 自动清理 2 天前的日志文件
- 用于问题排查和性能分析
- 可在界面上取消勾选关闭日志保存

## 常见问题

### Q: 启动后没有显示订阅地址？
A: 检查是否缺少 `nodes.txt` 文件，或端口被占用。

### Q: 扫描很久没有找到有效 IP？
A: 尝试提高延迟阈值（如 300ms），或切换到延迟更高的机房。

### Q: IP 频繁切换导致订阅不稳定？
A: 等待订阅 IP 自动锁定（刷新 10 次后），或降低延迟阈值筛选更稳定的 IP。

### Q: 如何查看 cfnat 的原始输出？
A: 勾选"保存 cfnat 日志"选项，日志保存在 `logs/` 目录下。

### Q: 程序提示"程序已在运行"？
A: 检查是否已经有一个实例在运行，或删除 `cfnat_sub.pid` 文件后重试。

## 安全提示

- ⚠️ IP 切换过于频繁可能导致账号风险
- ⚠️ 建议高峰期适当提高延迟阈值
- ⚠️ 关闭窗口时建议选择"关闭 cfnat"，避免后台残留进程
